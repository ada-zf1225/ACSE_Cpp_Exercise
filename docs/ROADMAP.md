# Roadmap

## Phase 1: Foundation (Week 1)
- [ ] Set up repository standards and CI
- [ ] Create first two practical solution folders
- [ ] Define labels and milestones in GitHub

## Phase 2: Delivery Rhythm (Week 2-4)
- [ ] Solve practicals with issue-driven workflow
- [ ] Open one PR per practical (or per major task)
- [ ] Keep PR notes focused on design and testing

## Phase 3: Quality and Reflection (Week 5+)
- [ ] Add stronger tests for edge cases
- [ ] Add performance notes where relevant
- [ ] Publish weekly progress summary in issues/discussions

## Definition of Done (per exercise)
- [ ] Code compiles locally
- [ ] Tests pass locally and in CI
- [ ] README for the exercise updated
- [ ] PR includes rationale and trade-offs
